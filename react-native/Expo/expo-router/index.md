Expo Router 是 Expo 生态下为 React Native 应用设计的**文件系统路由解决方案**，基于 React Navigation 构建，核心是通过文件/文件夹结构自动生成路由，无需手动配置路由表。


### 一、核心特性
1. **文件系统驱动路由**  
   应用根目录下的 `app` 文件夹是路由根目录，文件/文件夹名称直接对应路由路径。例如：
   - `app/index.js` → 根路径 `/`
   - `app/profile.js` → 路径 `/profile`
   - `app/posts/[id].js` → 动态路由 `/posts/:id`（`[id]` 为动态参数）

2. **跨平台一致性**  
   一套路由代码可同时运行在 iOS、Android 和 Web 端，无需为不同平台单独适配路由逻辑，自动处理平台特定的导航交互（如 Web 端的 URL 导航、移动端的栈导航）。

3. **深度集成 Expo 生态**  
   无缝衔接 Expo 核心工具，如 `expo start` 热重载、EAS Build（构建）、EAS Update（热更新），且支持 Expo 提供的原生功能（如相机、定位）在路由间传递状态。

4. **内置导航模式**  
   无需额外安装 React Navigation 插件，默认支持常见导航模式：
   - 栈导航（Stack）：用于页面跳转（如从列表页到详情页）
   - 标签导航（Tabs）：通过在 `app` 目录下创建 `_tabs` 子目录实现底部/顶部标签栏
   - 抽屉导航（Drawer）：通过 `_drawer` 目录实现侧边抽屉


### 二、核心工作原理
1. **路由映射规则**  
   Expo Router 在项目启动时，会扫描 `app` 目录下的所有文件，按照以下规则生成路由：
   - 普通文件（如 `page.js`）：对应一个路由页面，路径为文件相对 `app` 的路径（不含后缀）。
   - 特殊目录（如 `_tabs`、`_drawer`）：标记该目录下的路由使用对应导航模式，目录名本身不参与路径生成。
   - 动态文件（如 `[param].js`）：生成带参数的动态路由，参数可通过 `useLocalSearchParams` 钩子获取。

2. **导航实现方式**  
   通过 Expo Router 提供的 Hooks 和组件实现页面跳转：
   - `useRouter()`：获取路由实例，调用 `push('/path')`（新增页面）或 `replace('/path')`（替换当前页面）实现导航。
   - `useLocalSearchParams()`：获取当前路由的动态参数（如 `[id].js` 中的 `id`）。
   - `<Link href="/path">`：类似 Web 端的 `<a>` 标签，实现声明式导航，支持跨平台。


### 三、核心优势
1. **零配置成本**  
   无需手动编写路由配置文件（如 `routes.js`），仅通过调整 `app` 目录结构即可管理路由，降低新手学习成本。

2. **开发效率高**  
   支持热重载，修改 `app` 目录下的文件后，路由会自动更新，无需重启应用；同时提供清晰的目录结构，便于团队协作维护。

3. **原生体验好**  
   基于 React Navigation 底层实现，保留 React Native 原生的导航动画和交互逻辑（如 iOS 的侧滑返回、Android 的返回键响应）。

4. **Web 端友好**  
   自动同步 Web 端的 URL 与移动端的路由状态，支持浏览器前进/后退按钮，且无需额外配置即可实现 Web 端路由。
